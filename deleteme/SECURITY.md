# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, report them by opening a [GitHub Security Advisory](../../security/advisories/new).
This keeps the details private while we investigate and prepare a fix.

When reporting, please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant logs, screenshots, or proof-of-concept code

We'll acknowledge your report within 48 hours and aim to release a fix within 14 days for confirmed vulnerabilities, depending on severity and complexity.

## Scope

This policy applies to the KitOps Desktop application. Vulnerabilities in the
[KitOps CLI](https://github.com/kitops-ml/kitops) should be reported to that repository.

## Supported Versions

We only maintain the latest release. Please make sure you're running the most recent version before reporting a vulnerability.
